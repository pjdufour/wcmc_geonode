UNEP-WCMC Naming Standards

This document shows the rules for naming files (including GIS datasets), database tables and columns. There is a set of mandatory rules to be applied to all data and a set of non-mandatory rules. Prefixes and Suffixes are mainly non-mandatory, the only exception is the category code for GIS datasets. If information under prefixes and suffixes is used on a name the rules must be applied even if they are non-mandatory. All documents and datasets added to the Data Portal must follow these rules.

General Rules (File, Folder, Table and Column Names)


Mandatory Rules

  • They must be unique
  • They should be indicative of what the file contains
  • The characters used must be A..Z, 0..9 and underscore(_).
  • Dot (.) should be used only for delimiting file extension
  • Underscore is used as text delimiter
  • Spaces are not allowed
  • All characters must be lowercase
  • Name elements should be ordered from general to specific detail of importance as much as possible.
  • People names if needed should be in the format Surname + First Initial (e.g. BeckhamD, MessiL)
  • Prefixes and Suffixes are not mandatory (except for categories in GIS datasets) but if they are used their rules and order must be applied

Forbidden words

  • Test
  • Last
  • Final
  • Edit
  • Temp
  • Full name of author
  • Clean
  • Working
  • Annotated
  • Long file/folder names (more than 25 characters)
  • Copy
  • Version (just "v")

Suffixes (Non-Mandatory)


In terms of suffixes, all of them are non-mandatory. Timestamps should only be used for regular releases of data (e.g. WDPA). In the following table we list both of them, the order starts from the end of the filename to the beginning.


Order (Desc)

Type

Mandatory

1

Version Rules (see below)

2

Timestamp

Timestamp Rules (See below)



Versions

Suggested Versioning in the development process:

  • Must be the last element of the filename following a “v”
  • Must have a X-YY-ZZ format
  • X rules:
    • X is for major versions
    • 0 when the document is still being written/developed
    • 1 is for the first version without reviews
    • 2 is for final version candidate
    • 3 is for final version



  • YY is for changes inside an X version
  • ZZ is for minor fixes in a YY version
Timestamps
  • Dates should be ordered: YEAR, MONTH, DAY. (e.g. YYYYMMDD, YYYYMMDD, YYYYMM).
  • Time should be ordered: HOUR, MINUTES, SECONDS (HHMMSS)
  • More specific dates/time can be ignored (e.g. for a monthly release 201601 is enough)

Specific rules for tables

  • The reserved words in this list must be avoided.
  • Table names should be in plural as they list a group of objects(e.g. countries, users, names)

Column Names

  • The characters used must be A..Z, 0..9 and underscore(_).
  • 0..9 and underscore (_) cannot be used as first character of the column
  • Underscore is used as text delimiter
  • Spaces are not allowed
  • All characters must be lowercase
  • Foreign Key must be singular (e.g. country_id)

Examples

Starting Point

Imagine you produced a file with the following name:

"industries in burkina faso FEBruary 1886??? final final last_edited version6.5 joanna_felix_ag.doc.docx"

OK, This is just an example. Don't go to wikipedia to search for accuracy errors.

There is a lot to change in this file. Let's do it!

Meeting General Rules

There are several rules that are not satisfied in this name:

  • Not allowed characters: ??
    • "microchip industries in burkina faso FEBruary 1886 final final last_edited version6.5 joanna_felix_ag.doc.docx"
  • No Spaces
    • "microchip_industries_in_burkina_faso_FEBruary_1886_final_final_last_edited_version6.5_joanna_felix_ag.doc.docx"
  • Lower case characters: FEBruary
    • "microchip_industries_in_burkina_faso_february_1886_final_final_last_edited_version6.5_joanna_felix_ag.doc.docx"
  • Dot (.) should be used only for delimiting real file extension
    • "microchip_industries_in_burkina_faso_february_1886_final_final_last_edited_version6_5_joanna_felix_ag.docx"
  • Going from general to specific: industries in Burkina Faso
    • "burkina_faso_microchip_industries_february_1886_final_final_last_edited_version6_5_joanna_felix_ag.docx"
  • Naming format: Joanna Felix and ag (Alan Guiness) as prefixes
    • "felixj_guinessa_burkina_faso_microchip_industries_february_1886_final_final_last_edited_version6_5.docx"
  • Date format should be in YYYYMM format as suffix
    • "felixj_guinessa_burkina_faso_microchip_industries_final_final_last_edited_188602_version6_5.docx"

Use of forbidden words

The following forbidden words are used on this name:

  • Final
  • Last
  • Edit
  • Version (just "v")
"felixj_guinessa_burkina_faso_microchip_industries_188602_v6_5.docx"

Not Mandatory Changes

As it is, our name already is already fulfilling mandatory rules but it can be improved.

Country code

Instead of writing down Burkina Faso people can just write this country's ISO3 code:

bfa_felixj_guinessa_microchip_industries_188602_v6_5.docx
Category code

We can also add the category code so it is sortable in an easier way:

05_bfa_felixj_guinessa_microchip_industries_188602_v6_5.docx
Versioning

Let's assume that version 6.5 is being is a final version candidate that has been edited for minor changes by ALan Guiness :

05_bfa_felixj_guinessa_microchip_industries_188602_v2_0_1.docx

Final Result

So, from a file named as:

"industries in burkina faso FEBruary 1886??? final final last_edited version6.5 joanna_felix_ag.doc.docx"

After applying all the rules we got:

05_bfa_felixj_guinessa_microchip_industries_188602_v2_0_1.docx

Short name and easy understandable
