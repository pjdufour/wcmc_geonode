Documents

Despite not being its main focus, UNEP-WCMC Geonode is an excellent place to store documents and search for them.

The Documents tab also allows attaching information to Geographic layers when metadata fields do not have all the information needed

Browsing and Searching

All documents can be browsed and searched by clicking on the Documents tab. You will be able to filter results by Text, Type, Categories, Keywords, Owners, Date, Regions and Extent.

Additional Metadata for Geospatial Layers

When you need to attach documents to datasets you can use the link to option to find the dataset you have just added to UNEP-WCMC Geonode. In this case you will need to provide information about this attached document in the metadata fields (which should be different from the Metadata fields for the entire dataset). You should follow the instructions below to add your files to the system.

Adding Documents

The process of uploading document depends on if the data was produced in the centre or if it has external sources.

The workflow on importing GIS datasets is described on the following diagrams both for produced internally data and externally sourced:

Produced Data External Data

The workflow is similar but the license to use depends on the source for external data and on you or the Data Champion of your programme for produced data.

The steps are: Renaming According to Naming Standards, Uploading Files, License and Permissions Management, Adding Metadata.

Renaming According to Standards

Before uploading the file you must check if both its name and its attribute table comply with UNEP-WCMC naming standards.

Uploading Files

Depending on the file size it can take some time to upload. While you wait you can set the permissions for the dataset. If you have multiple files you should compress them all together.

License and Permissions Management

If you do not know which License to use, please contact the Data Champion of your programme. If you already know which License applies to your data, you can set up the permissions on the right side of the page. If you think that License has not been used before in UNEP-WCMC you will also need to contact the Data Champion of your programme so he/she can add it to the Licenses list.

Adding Metadata

This is the most important and also the most time-consuming task. Information about the different fields is provided when you mouseover each box.

If there is other information about the dataset that might be relevant you should add as a document linked to this dataset. To do that fill in the metadata and then follow the instruction about adding documents to UNEP-WCMC Geonode

Owner

Name of the person responsible for this dataset and its metadata

Title

Name by which the cited resource is known. Should be concise and easily searchable

Date

Date when the cited resource was produced (it can be an estimate). If there is no information provided, it should be the date when the resource was published or revised.

Date Type

Identification if the date above refers to Publication, Creation or Revision.

Edition

If the dataset is part of a series (e.g. WDPA) this field should be used for the edition. Not to be used for versioning.

Abstract

Brief narrative summary of the content of the resource(s).

Purpose

Summary of the intentions with which the resource(s) was developed.

Maintenance Frequency

Frequency with which modifications and deletions are made to the data after it is first produced.

Region

If the document refers to a spatial extend. Global/Region/Country

Restrictions / Other Restrictions

Limitation(s) placed upon the access or use of the data

License

License of the document. If it is not listed please contact the Data Champion of your programme

Language

Language used within the document

Spatial Representation Type

Format of the information used to represent a Geographical document. Not to be used for non spatial documents.

Temporal extent (start and end)

Time period covered by the content of the document

Supplemental Information

Supplemental Information should be added when relevant. Should be included:

    DOI
    Original File Name
    wcmc.io link

Distribution URL

URL automatically defined by geonode with a link to the resource page

Distribution Description

Detailed text description of what the online resource is/does

Data Quality Statement

General explanation of the data producer's knowledge about the lineage of the document stating the level of reliability it has.

Featured

Check if this resource should be advertised in the homepage

Is Published

Check if this resource should be published and searchable

Thumbnail url

Link to the image used as thumbnail for the resource (automatically set by UNEP-WCMC Geonode)

Keywords

Commonly used word(s) or formalised word(s) or phrase(s) used to describe the subject (space or comma-separated).

Tags Type Mandatory
Scale (e.g. Regional, Global, Country) String, Text, ISO3 Codes
Theme (e.g. Bluecarbon, CITES, habitats) String, Text ✓
Output (e.g. Brochure, Report, Tech Brief, Infographic) String, Text ✓
Client (e.g. GEF, IUCN, internal) String, Text ✓
Partners String, Text ✓
Provider (e.g. IUCN, Microsoft) String, Text
Format (e.g. xls, csv, docx, tiff) String, Text

Point of Contact

Person to be contacted regarding this resource.

Metadata Author

Person that has authored the metadata

Link to:

To be used when the document refers do a Geographic Dataset

Category

High-level geographic data thematic classification to assist in the grouping and search of available documents.

Final Control

After all this work your dataset is now on UNEP-WCMC Geonode! You can now try to find browsing on the Layers tab or Searching for it
